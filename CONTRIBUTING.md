# Contributing to Loadoutz.io

Thank you for your interest in contributing.

## What We're Looking For

- Documentation improvements
- Bug fixes
- Integration examples
- Security enhancements

## What We're Not Looking For

- New features without prior discussion
- Architectural changes
- Breaking changes to existing functionality

## Getting Started

1. Fork the repository
2. Create a branch for your work (`git checkout -b feature/description`)
3. Make your changes
4. Ensure tests pass (if applicable)
5. Submit a pull request

## Documentation

If you're improving documentation:
- Keep language clear and factual
- Avoid marketing language
- Focus on accuracy and clarity
- Update relevant sections only

## Code Style

- Follow existing patterns in the codebase
- Keep changes minimal and focused
- Add comments only when necessary to explain non-obvious logic
- Run linting/typechecking before submitting (if applicable)

## Security

- Never commit secrets or credentials
- Report security issues via email (see SECURITY.md)
- Follow OAuth and token handling best practices

## Pull Request Process

1. Describe what your PR does and why
2. Link to related issues (if any)
3. Ensure your description is clear and concise
4. Wait for review and feedback

## Questions?

Open an issue with the "question" label.
